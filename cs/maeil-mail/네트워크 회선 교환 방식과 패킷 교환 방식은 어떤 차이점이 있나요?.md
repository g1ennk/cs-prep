# 네트워크 회선 교환 방식과 패킷 교환 방식은 어떤 차이점이 있나요?

## 요약

###   

1. 회선 교환 방식
    - 특정 사용자를 위한 회선의 경로를 미리 설정하고 이 경로를 이용해서 호스트끼리 메시지를 주고 받는 방식을 의미한다.
    - 해당 방식은 미리 회선을 설정한다는 점에서 주어진 시간 동안에 전송되는 데이터의 양이 비교적 일정하고 안정적이다.
    - 단, 회선 이용 효율이 떨어진다는 단점이 존재한다.
    - 대표적인 회선 교환 방식 사례는 유선 전화망이 있다.
    - 결론: 전용선을 깔고 통화 -> 통화 동안 대역폭 예약하여 품질이 안정적이고 대신 사용하지 않은 시간에도 회선이 묶여 비용과 자원 낭비
2. 패킷 교환 방식
    - 목적지를 정해두고 메시지를 패킷으로 분할해서 보내고, 목적지에서 패킷을 조립해서 확인하는 방식이다.
    - 해당 방식에서 라우터는 주어진 패킷을 최적 경로로 전달하는 핵심적인 역햘을 수행한다.
    - 이러한 특성으로 경로는 수시로 변경될 수 있고, 데이터를 전송하는 동안에만 네트워크 자원을 사용한다는 점에서 회선 교환 방식과 차이가 있다.
    - 회선 교환 방식과 반대로 회선 이용 효율이 높지만, 경로 탐색에서 지연이 발생하거나, 패킷을 위한 헤더로 오버헤드 발생 가능성이 존재한다.
    - 결론: 도르를 같이 쓰는 차량 -> 데이터를 패킷으로 쪼개 공유망에 실어 보내 망 활용도, 확장성, 비용 절감에 유리하지만, 지연 및 손실 가능성 존재

### 비교표

| 구분    | **회선 교환(Circuit Switching)** | **패킷 교환(Packet Switching)**               |
|-------|------------------------------|-------------------------------------------|
| 연결 방식 | 통화 전용선처럼 **통로를 미리 고정**(호 설정) | 데이터를 **작은 패킷**으로 쪼개 **공유망**에 흘려보냄         |
| 자원 점유 | 통화 동안 **대역폭 고정 점유**          | 트래픽 순간에만 **공유 사용**(통계적 다중화)               |
| 지연/품질 | **지연 일정, 지터 거의 없음**, 손실 적음   | **가변 지연/지터**, 혼잡 시 손실·재전송 가능              |
| 효율    | 유휴 시간에도 회선이 묶여 **비효율**       | 다수가 나눠 써 **고효율/저비용**                      |
| 설정 시간 | **호 설정 필요**(초 단위)            | **즉시 전송**(연결 설정 불필요, UDP 기준)              |
| 신뢰/순서 | 고정 경로라 **순서 보장**             | 경로·순서 **보장 안 됨**(IP 기본). 상위 프로토콜이 보완(TCP) |
| 대표 예  | 전통 전화망(PSTN), ISDN           | 인터넷(IP), 이더넷, VoIP, 스트리밍                  |

## 발화

회선 교환 방식은 세션 동안 대역폭을 독점으로 예약하여 데이터 전송량이 일정하고 안정적이지만, 회선 이용 효율이 떨어진다.

반면에 패킷 교환 방식은 망을 공유하여 사용하기 때문에 효율과 확정성은 좋지만, 지연과 손실 가능성이 존재합니다.
