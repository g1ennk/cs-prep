# 서버리스란 무엇인가요

## 요약

### 정의

- 서버리스(Serverless)란 클라우드 업체에서 직접 인프라를 관리하고 동적으로 크기를 조정하면서 유지 관리하는 방식을 의미한다.
- AWS를 사용해도 OS, 파일 시스템 관리 등은 직접 인프라 담당자나 개발자가 해야하는데, 이를 모두 AWS에서 관리하는 것을 말한다.
- 서버리스를 도입한다면 직접 서버를 관리할 필요 없이 유연하게 확장하고 사용하지 않은 용량에 대해서는 비용을 지불하지 않을 수 있다는 장점이 있다.
- 대표적으로 AWS Lambda, S3, SQS가 있다.

### 장점

1. 서버 운영 및 관리 부담 없음
    - 운영 작업을 모두 클라우드가 수행
2. 이벤트 기반 실행
    - 함수가 특정 이벤트에 의해 필요할 때만 실행 ex) AWS Lambda, GCP Cloud Functions
3. 사용한 만큼만 비용 지불
    - 서버를 계속 킬 필요 없이 요청에 따라 비용 청구
4. 자동 스케일링

### 단점

1. 콜드 스타트
2. 장시간 실행에는 적합하지 않음
3. 벤더 종속성 높음

### 서버리스 아키텍처에서 백엔드 코드는 어떻게 개발하나요?

- 백엔드에선 FaaS(Function as a Service)를 활용해 개발할 수 있다.
    - FaaS란 특정 이벤트가 발생했을 때 함수가 실행되는 방식을 의미한다.
    - 예를 들어, `GET /hello`라는 경로에 이벤트가 발생하면 이를 처리하는 함수가 실행되어 응답한다.
    - 이로 인해 개발자는 비즈니스 로직을 작성하고, 클라우드 인프라에 코드 조각(함수)를 배포하면 된다.
- BaaS(Backend as a Service)도 활용 가능하다.
    - BaaS는 AWS와 같안 클라우드 서비스에서 만든 인증, 소셜 서비스 등의 완성된 백엔드 기능을 사용할 수 있다.
    - 예를 들어, Firebase Authentication을 활용하면 비교적 적은 코드로 로그인 기능을 활용할 수 있다.

### 실무 패턴

1. 백엔드 API 일부만 서버리스로 분리 ex) 이미지 처리, 알림 기능
2. 크론 작업
3. Webhook 처리
4. 초기 스타트업 MVP 전체
5. 사내 자동화
